提出一个「分层式人物拼图」 设计方案，既保留拼图的群像概览，又融入名片卡的视觉仪式感。

设计方案：分层式人物拼图
第一层：群像拼图概览（方案B核心）
顶部区域 - 群像摘要卡

内容逻辑：

共同点：显示出现频率最高的3个兴趣标签

人群构成：用进度条显示各社交原型占比，让用户快速理解桌上能量分布

视觉与交互规范
布局结构

移动端：2列栅格，名片卡等高对齐

桌面端：3-4列响应式布局

间距：卡间间距≥12px，确保触控区域

视觉层次

css
/* 颜色系统 */
主标签色：#主题色（随桌变化）
兴趣标签：#666（中性灰）
次要信息：#999（浅灰）
背景色： #f8f9fa

/* 标签样式 */
角色标签：实心背景 + 白色文字
兴趣标签：描边边框 + 中性色文字
共同点指示器：实心圆点（主色）/空心圆点（灰色）
交互逻辑

默认状态：显示所有名片正面

点击名片：单个翻转，显示反面详细信息

翻转后：显示「共同点徽章」和个性化描述

二次点击：翻转回正面

信息架构
数据映射关系

text
用户数据 → UI组件
name_display → 昵称
archetype → 角色标签 + 图标
interests_ranked_top3 → 兴趣徽章（前3个）
age_band/industry → 次要信息（反面）
匹配算法数据 → 共同点计数
内容生成规则

共同点计数：基于兴趣重叠数（1-3+个）

个性化描述：从用户填写的兴趣中提取关键词生成短句

人群构成：基于本桌所有用户的archetype分布计算

技术实现要点
渐进式加载

先加载群像摘要和名片正面

按需加载名片反面详细信息

图片懒加载 + 骨架屏占位

动画效果

名片翻转：3D transform，300ms缓动

共同点计数：数字递增动画

群像进度条：宽度渐增动画

空状态与异常处理
数据不足时

隐私保护

严格遵循用户的可见性设置

隐藏信息的位置用统一的占位文案：「信息暂不可见」

这个方案既提供了方案B的「群像概览」，帮助用户快速理解桌上的人群构成；又保留了方案E的「名片仪式感」，通过翻转交互让了解每个桌友的过程更有趣味性。所有设计都确保在预览阶段不涉及破冰环节，纯粹是信息展示。
